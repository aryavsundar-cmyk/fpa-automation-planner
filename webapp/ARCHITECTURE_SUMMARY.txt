RECOMMENDATION SUMMARY
======================

1. FRAMEWORK STRATEGY: Hybrid Vanilla + Vue.js
   ✓ Phases 1-3: Stay vanilla JS (590 hrs) — zero overhead, fast iteration
   ✓ Phase 4+: Introduce Vue.js 3 (400 hrs) — form complexity, API integration
   ✓ Phases 4-5: Migrate to Pinia state management
   
   IMPACT: +1,200 LOC (Vue), +400 LOC (refactoring), zero disruption to Phase 1-3

2. STATE MANAGEMENT: Custom Manager → Pinia Migration
   ✓ Phases 1-3: Lightweight StateManager class (300 LOC)
     - Simple pub/sub pattern: subscribe(key, callback)
     - Immutable updates: setState(key, value)
     - Scenario management: createScenario(), switchScenario()
   
   ✓ Phase 4: Migrate to Pinia (standard Vue.js store)
     - 80% functionality, 20% complexity vs. Redux
     - Composition API: better code organization
     - Devtools integration for debugging
   
   IMPACT: Manageable complexity for 5+ scenarios + multiple views

3. STYLING: Continue Tailwind v4
   ✓ Keep existing tailwind.config.js (already optimized)
   ✓ No migration needed — v4 is latest, performant
   ✓ Optional: Pre-compile CSS when Phase 4 Vite build is added
   
   IMPACT: Zero technical debt, leverages existing setup

4. VISUALIZATION LIBRARIES: Chart.js + D3.js
   ✓ Phase 2: Chart.js only (bar, line, pie — 60KB)
   ✓ Phase 3: Add D3.js (custom viz, dependency maps — 200KB → 100KB tree-shaken)
   ✓ Phase 4+: Optional Plotly.js for advanced dashboards (skip if time-limited)
   
   IMPACT: 260KB combined, covers 95% of visualization needs

5. DATA PERSISTENCE: localStorage + Optional Backend Sync
   ✓ Phases 1-3: localStorage only (5-10MB quota, offline-first)
     - Max 5 scenarios × 500KB each = 2.5MB used
   
   ✓ Phase 4: Optional Express.js endpoints
     - POST /api/scenarios/save (multi-device sync)
     - GET /api/scenarios/:userId (backup/recovery)
   
   IMPACT: No database required Phases 1-3; scalable if needed Phase 4+

6. EXPORT FUNCTIONALITY: PDF + Excel
   ✓ Phase 3: jsPDF + html2canvas (PDF, 70KB)
   ✓ Phase 3: SheetJS (Excel export, 150KB)
   ✓ Phase 4+: Optional CSV + JSON (trivial)
   
   IMPACT: Covers all proposal/scenario export requirements

7. CLAUDE API INTEGRATION: Server-Side Wrapper
   ✓ Phase 4: Backend Express.js endpoint
   ✓ POST /api/generate-proposal (scenario config → AI narrative)
   ✓ Protects API key, enables caching, rate limiting
   
   IMPACT: Secure, scalable, 200 LOC in server.js

8. BUILD STRATEGY: No Build Phases 1-3, Vite Phase 4+
   ✓ Phases 1-3: Vanilla JS modules + ES6 imports (no build)
   ✓ Phase 4: Introduce Vite (fastest build tool, Vue.js standard)
   ✓ Code splitting: phase1, phase2, phase3, phase4, vendor, viz, export
   
   IMPACT: Fast iteration early, optimized bundling later

CRITICAL SUCCESS FACTORS
========================

1. Week 1-2 Refactoring (Highest Risk)
   - Must maintain 100% backward compatibility
   - Test each extracted module independently
   - Mitigation: Small commits, frequent testing, parallel feature development

2. State Management (Medium Risk)
   - Scenario save/load must be bulletproof
   - Test with 5+ concurrent scenarios
   - Mitigation: Unit tests for StateManager, localStorage quota testing

3. Visualization Library Conflicts (Medium Risk)
   - D3 + Chart.js DOM contention (both mutate DOM)
   - Mitigation: Isolate in separate containers, test integration early (Week 6)

4. Vue.js Migration (Low Risk)
   - Phase 4 adoption is self-contained
   - Can skip if schedule slips (app works in vanilla)
   - Mitigation: Build Vue components in isolation, keep vanilla fallback

5. Claude API Integration (Low Risk)
   - Rate limiting, caching strategy
   - Mitigation: Server-side queue, localStorage cache (5-min TTL)

RISK MITIGATION STRATEGIES
============================

  Mitigation: Refactor into modules immediately (Week 1-2)

  Mitigation: Custom StateManager tested with unit tests, Pinia by Phase 4

Risk 3: localStorage quota exceeded
  Mitigation: Max 5 scenarios, compression, optional backend sync (Phase 4)

  Mitigation: Separate DOM containers, test integration early (Week 6)

  Mitigation: Server-side caching (5-min TTL), request queue, lower token limits

  Mitigation: Build Vue in isolation, keep vanilla.js fallback until Phase 4 stable

Risk 7: Mobile responsiveness issues
  Mitigation: Test on real devices (iPhone 375px, iPad 768px) every iteration
